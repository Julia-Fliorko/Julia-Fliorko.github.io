Customer Churn Analysis

This project prepares a customer dataset for churn analysis. The work focuses on cleaning, validating, and documenting the data so it can be used reliably in later analysis or modeling.

The raw data included missing values, inconsistent formats, mixed data types, and fields containing personally identifiable information (PII). I standardized missingness, corrected types and categories, removed PII, and created derived variables such as tenure, order frequency, and spend metrics.

Output: a cleaned dataset for churn analysis, supporting visualizations, and notebooks that document the steps and decisions made during preprocessing.

Problem

Churn analysis depends on consistent definitions and clean data. Before building any churn model, the dataset needs to be structured, validated, and free of issues that can bias results.

This project focuses on dataset preparation: removing PII, standardizing formats and missing values, validating key fields, and creating stable derived features to support later churn analysis.

Dataset

Primary Dataset

Customer-level records with behavioral and transactional variables (for example: tenure, orders, spend, and a churn label).

The dataset required preprocessing due to missing values, inconsistent formats, and mixed data types.

Why it matters

Data issues can distort churn results (for example: inconsistent definitions, leakage, or unhandled missingness).

The goal is to produce a dataset that is suitable for analysis and modeling and is clearly documented.

Approach

1) PII Handling & Governance

Removed personally identifiable information (PII) and documented the fields excluded from the final dataset.

2) Data Cleaning & Consistency Checks

Standardized missing values, corrected data types, and applied consistency checks (ranges, categories, and basic validity rules).

3) Exploratory Data Analysis

Reviewed distributions and outliers to understand the data and to identify issues before feature engineering.

4) Feature Engineering + Export

Created derived variables (for example: tenure, order frequency, and spend metrics) and exported a consolidated dataset for later analysis or modeling.

Results

  • Standardized dataset: missing values, formats, and types were cleaned to improve consistency.
  • PII removed: sensitive identifiers were excluded to support safe sharing and analysis.
  • Derived features added: tenure, frequency, and spend variables were created using consistent logic.
  • Scope: the work covers data preparation and exploratory summaries, not churn model performance.
