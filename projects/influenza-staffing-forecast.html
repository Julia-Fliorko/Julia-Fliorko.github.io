Influenza Staffing Forecast

This project turns influenza mortality trends into staffing intelligence for healthcare planning. The objective is to identify when and where demand spikes, so medical teams can be scheduled proactively instead of reacting mid-season.

Using Excel-based analysis and Tableau delivery, the workflow translates raw time patterns into stakeholder-ready signals‚Äîpeak timing, geographic variability, and vulnerability prioritization‚Äîto support policy and resource allocation.

Project Overview

Key Questions

  • When do influenza deaths peak, and how consistent is peak timing across years?
  • Which states show stable seasonal peaks vs. unpredictable fluctuations?
  • Which population segments face the highest burden, and how should that affect staffing plans?
  • How can limited staff be allocated efficiently across locations and time windows?

Quick Facts

  • Method: Excel analytics (trend profiling, peak-month detection, ranking logic)
  • Output: KPI tables + peak-month map + prioritization views
  • Delivery: Tableau dashboard + Excel extracts

Problem

Healthcare staffing decisions often rely on a generic ‚Äúflu season‚Äù assumption, but influenza burden varies by state, timing, and population vulnerability. That misalignment creates operational risk: staffing arrives too late in early-peak regions and gets over-allocated where peaks are milder or inconsistent.

This project converts influenza mortality patterns into decision-ready metrics so stakeholders can plan staffing based on peak windows, regional variation, and priority need‚Äînot guesswork.

Dataset

Source

Influenza mortality and population data structured by state, month, and year, with demographic breakdowns used to assess differential burden across groups.

The dataset supports month-over-month comparisons and year-over-year consistency checks.

Notes

Outputs are formatted as reusable Excel tables that feed directly into Tableau for stakeholder review. Standardized state/time fields enable consistent aggregation and peak detection.

The analysis focuses on policy and resource allocation use-cases, not exploratory ‚Äúdashboard-only‚Äù visuals.

Approach

1) Data Preparation (Excel)

Cleaned and standardized fields for state and time (month/year). Validated totals and built analysis-ready tables to support pivoting, aggregation, and consistent comparisons.

2) Peak Detection

Identified peak months per state (and per year when applicable), then summarized across years to determine each state‚Äôs most common peak month and how stable that pattern is.

3) Prioritization Logic

Created ranking and tiering logic (low/medium/high need) to support staffing decisions when resources are constrained‚Äîso allocation is defensible and consistent.

4) Stakeholder Delivery

Packaged outputs into Tableau views designed for fast decision-making: geographic peak timing, trend context, and prioritization signals in one place.

Results

  • Peak timing varies by state: a single national timeline hides meaningful regional differences and can lead to staffing misalignment.
  • Predictability differs across locations: stable patterns support scheduling, while volatile patterns require contingency planning.
  • Prioritization needs more than timing: staffing decisions should reflect both peak windows and relative burden/vulnerability.
  • Decision-ready outputs: tables and visuals provide measurable justification for staffing and allocation discussions.
